Как сделать Indent в CodeMirror

ChatBotChatApp

Вы хотите изменить ширину отступа слева в CodeMirror 5 с двух пробелов на три. Это достигается путем настройки параметра indentUnit.

Вот как вы можете это сделать:

1. При инициализации CodeMirror:

Когда вы создаете экземпляр CodeMirror, вы можете передать опции, включая indentUnit.

var myCodeMirror = CodeMirror(document.getElementById("my-code-mirror-div"), {
    value: "ваш код здесь",
    mode: "javascript", // или любой другой режим
    indentUnit: 3 // Устанавливаем отступ в 3 пробела
});

2. Если CodeMirror уже инициализирован:

Если у вас уже есть существующий экземпляр CodeMirror, вы можете изменить indentUnit с помощью метода setOption.

// Предполагая, что у вас уже есть экземпляр myCodeMirror
myCodeMirror.setOption("indentUnit", 3);

Как сделать extraKeys для Ctrl + Shift + \ и прицепить вызов самодельной функции

ChatBotChatApp

1. Как сделатьextraKeys для Ctrl + Shift + \и прицепить вызов самодельной функции:

Сочетание Ctrl + Shift + \ будет выглядеть так: "Cmd-Shift-\\" или "Ctrl-Shift-\\"

Пример:

extraKeys: {
    "Cmd-Shift-\\": "yourFunctionName", // Или "Cmd-Shift-\\": myFunction
    "Ctrl-Space": "autocomplete"
}

2. Прицепить вызов самодельной функции myFunction():

Вы можете передать в extraKeys как строку, так и саму функцию (или ее имя, если она находится в глобальной области видимости).

Вариант 1: Передача имени функции (если myFunction в глобальной области):

function myFunction() {
    console.log("Моя самодельная функция вызвана!");
    // Здесь ваш код
}

var editor = CodeMirror(document.getElementById("my-code-mirror-div"), {
    // ...
    extraKeys: {
        "Cmd-Shift-\\": "myFunction", // Передаем имя функции как строку
        "Ctrl-Space": "autocomplete"
    }
});

Вариант 2: Передача функции напрямую (более предпочтительно, если функция определена локально):

var editor = CodeMirror(document.getElementById("my-code-mirror-div"), {
    // ...
    extraKeys: {
        "Cmd-Shift-\\": function(cm) { // Получаем экземпляр CodeMirror как аргумент
            myFunction(cm); // Вызываем вашу функцию
        },
        "Ctrl-Space": "autocomplete"
    }
});

function myFunction(cm) { // Можно передать экземпляр редактора
    console.log("Моя самодельная функция вызвана!");
    console.log("Текущий редактор:", cm);
    // Здесь ваш код
}

Почему передача функции напрямую предпочтительнее:

  • Инкапсуляция: Ваша функция myFunction не загрязняет глобальное пространство имен.
  • Передача контекста: Вы можете передать экземпляр CodeMirror (cm) в вашу функцию, что позволит ей работать с редактором (например, вставлять текст, получать текущее значение и т.д.).

Полный пример:

// Определяем самодельную функцию, которая может принимать экземпляр редактора
function handleMyCustomAction(editorInstance) {
    console.log("Вызвана самодельная функция для Ctrl+Shift+\\.");
    console.log("Текущий текст в редакторе:", editorInstance.getValue());

    // Пример: вставить какой-то текст в позицию курсора
    var cursor = editorInstance.getCursor();
    editorInstance.replaceRange("<!-- Вставлено самодельной функцией -->", cursor);
}

var editor = CodeMirror(document.getElementById("my-code-mirror-div"), {
    value: `
<!DOCTYPE html>
<html>
<head>
    <title>Test</title>
</head>
<body>
    <p>Some content.</p>
</body>
</html>`,
    mode: "htmlmixed",
    theme: "eclipse",
    lineNumbers: true,
    indentUnit: 3,
    indentWithTabs: false,
    extraKeys: {
        // Сочетание клавиш: Ctrl/Cmd + Shift + \
        "Cmd-Shift-\\": function(cm) { // Принимаем экземпляр редактора
            handleMyCustomAction(cm); // Вызываем нашу функцию, передавая экземпляр
        },
        "Ctrl-Space": "autocomplete" // Стандартное автодополнение
    },
    gutters: ["CodeMirror-lint-markers", "CodeMirror-foldgutter", "CodeMirror-linenumbers"], // Проверьте правильность имени 'foldgutter'
    lint: true,
    foldGutter: {
        rangeFinder: CodeMirror.fold.auto
    }
});

Итого:

  • Для Ctrl + Shift + \ используйте "Cmd-Shift-\\".
  • Передавайте вашу функцию либо напрямую как анонимную функцию, которая вызывает вашу самодельную функцию, либо как имя функции, если она в глобальной области. Настоятельно рекомендуется передавать экземпляр редактора (cm) в вашу функцию для удобства работы с ним.

ChatBotChatApp

Конечно! Давайте покажем оба варианта: глобальную и локальную функцию, применительно к вашему случаю с extraKeys в CodeMirror.

Представим, что вы хотите, чтобы при нажатии Ctrl + Shift + \ выполнялась какая-то ваша самодельная логика.

Вариант 1: Глобальная функция

В этом случае myFunction определяется в глобальной области видимости (например, прямо в <script> теге, но вне других функций или классов, или в отдельном <script> файле, который загружается первым).

HTML (index.html):

<!DOCTYPE html>
<html>
<head>
    <title>Global Function Example</title>
    <link rel="stylesheet" href="lib/codemirror.css">
    <style>
        .CodeMirror { height: 200px; border: 1px solid #ccc; }
    </style>
</head>
<body>
    <div id="my-code-mirror-div"></div>

    <script src="lib/codemirror.js"></script>
    <!-- ... другие скрипты CodeMirror ... -->
    <script src="mode/htmlmixed/htmlmixed.js"></script>
    <!-- ... -->

    <!-- === ГЛОБАЛЬНАЯ функция === -->
    <script>
        // Эта функция определена в глобальной области видимости
        function myGlobalFunction() {
            console.log("Вызвана ГЛОБАЛЬНАЯ функция!");
            // Здесь может быть какая-то общая логика, доступная отовсюду
            // Например, взаимодействие с другими частями вашего приложения
        }
    </script>

    <!-- === Скрипт инициализации CodeMirror === -->
    <script>
        document.addEventListener("DOMContentLoaded", function() {
            var editor = CodeMirror(document.getElementById("my-code-mirror-div"), {
                value: "<div><p>Hello</p></div>",
                mode: "htmlmixed",
                lineNumbers: true,
                extraKeys: {
                    // Передаем имя глобальной функции как строку
                    "Cmd-Shift-\\": "myGlobalFunction",
                    "Ctrl-Space": "autocomplete"
                }
            });
        });
    </script>
</body>
</html>

Как это работает:

  • myGlobalFunction доступна напрямую в window объекте.
  • extraKeys принимает строку "myGlobalFunction". CodeMirror ищет функцию с таким именем в глобальной области и вызывает ее при нажатии Ctrl+Shift+\.
  • Недостаток: Если у вас много таких функций, глобальное пространство имен может засориться. Также, нет прямого способа передать экземпляр редактора cm в myGlobalFunction без дополнительных ухищрений.

Вариант 2: Локальная функция (предпочтительный)

В этом случае myLocalFunction определяется внутри области видимости вашего скрипта инициализации, и вы можете передать экземпляр редактора cm прямо ей.

<!DOCTYPE html>
<html>
<head>
    <title>Local Function Example</title>
    <link rel="stylesheet" href="lib/codemirror.css">
    <style>
        .CodeMirror { height: 200px; border: 1px solid #ccc; }
    </style>
</head>
<body>
    <div id="my-code-mirror-div"></div>

    <script src="lib/codemirror.js"></script>
    <!-- ... другие скрипты CodeMirror ... -->
    <script src="mode/htmlmixed/htmlmixed.js"></script>
    <!-- ... -->

    <!-- === Скрипт инициализации CodeMirror === -->
    <script>
        document.addEventListener("DOMContentLoaded", function() {

            // === ЛОКАЛЬНАЯ функция ===
            // Эта функция определена внутри области видимости, где создается редактор.
            // Она может принимать экземпляр редактора (cm) как аргумент.
            function myLocalFunction(editorInstance) {
                console.log("Вызвана ЛОКАЛЬНАЯ функция!");
                console.log("Редактор, который ее вызвал:", editorInstance);

                // Можно использовать экземпляр редактора для работы с кодом
                var currentContent = editorInstance.getValue();
                var cursorPosition = editorInstance.getCursor();
                console.log("Текущий контент:", currentContent);
                console.log("Позиция курсора:", cursorPosition);

                // Пример: вставить текст
                editorInstance.replaceRange("<!-- Local function inserted -->", cursorPosition);
            }

            var editor = CodeMirror(document.getElementById("my-code-mirror-div"), {
                value: "<div><p>Hello</p></div>",
                mode: "htmlmixed",
                lineNumbers: true,
                extraKeys: {
                    // Передаем анонимную функцию, которая вызывает нашу локальную функцию
                    // и передает ей экземпляр редактора (cm)
                    "Cmd-Shift-\\": function(cm) {
                        myLocalFunction(cm); // Вызываем локальную функцию с экземпляром редактора
                    },
                    "Ctrl-Space": "autocomplete"
                }
            });
        });
    </script>
</body>
</html>

Как это работает:

  • myLocalFunction определена внутри обработчика DOMContentLoaded. Она не “засоряет” глобальное пространство.
  • В extraKeys мы передаем анонимную функцию function(cm) { ... }. Эта функция будет вызвана CodeMirror при нажатии комбинации клавиш, и cm будет содержать экземпляр текущего редактора.
  • Внутри этой анонимной функции мы вызываем myLocalFunction(cm), передавая ей экземпляр редактора. Это дает вашей функции полный доступ к API CodeMirror.

Почему локальная функция лучше:

  1. Инкапсуляция: Код остается более организованным, переменные и функции не конфликтуют с другими частями вашего приложения.
  2. Передача контекста: Легко передавать editorInstance (именно так обычно называют экземпляр CodeMirror) в вашу функцию. Это критически важно, если ваша функция должна манипулировать содержимым редактора, получать его состояние и т.д.
  3. Поддержка: Легче понять, где определяется и используется функция, и как она взаимодействует с другими частями кода.

В большинстве случаев, когда вы пишете код для конкретного элемента, как ваш редактор CodeMirror, локальные функции являются предпочтительным подходом.
