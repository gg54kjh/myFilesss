Что я могу сделать для обхода Trusted Types как пользователь, на стороне клиента? Пошагово пожалуйста.

19 минут назад

Для обхода
Trusted Types на стороне клиента как пользователь (например, если сайт «сломан» или вы проводите тестирование), вы можете использовать несколько методов — от установки расширений до изменения параметров запуска браузера. 

Вариант 1. Использование расширений (Самый простой)
************** 
Этот метод позволяет временно отключать заголовки безопасности, которые активируют Trusted Types. 
  1. Установите расширение для управления CSP: Например, Disable Content-Security-Policy или аналогичное.
  2. Активируйте расширение: Перейдите на проблемный сайт. Нажмите на иконку расширения в панели инструментов браузера. Обычно иконка меняет цвет (например, с серого на цветной), сигнализируя об активации.
  3. Перезагрузите страницу: Теперь браузер будет игнорировать заголовок require-trusted-types-for, и блокировка исчезнет.
    • Внимание: Отключайте расширение после использования, так как это снижает общую безопасность браузера. 
Вариант 2. Запуск браузера без проверок безопасности
**************
Этот способ радикален и отключает многие механизмы защиты, поэтому используйте его только в изолированном окне. 
  1. Закройте все окна Chrome/Edge через Диспетчер задач.
  2. Запустите через терминал/командную строку с флагом --disable-web-security:
    • Windows: Нажмите Win + R, введите:
      chrome.exe --user-data-dir="C:/temp_chrome_user" --disable-web-security
    • macOS: В Терминале:
      open -na "Google Chrome" --args --user-data-dir="/tmp/chrome_dev" --disable-web-security.
  3. Результат: Браузер запустится с предупреждением о снижении безопасности, но Trusted Types и CORS не будут блокировать действия на сайтах. 
Вариант 3. Пользовательские скрипты (Tampermonkey)
**************
Если вы хотите исправить конкретный сайт, не отключая защиту полностью, можно создать «политику по умолчанию». 
  1. Установите расширение Tampermonkey.
  2. Создайте новый скрипт и вставьте в него код для создания пустой политики «default»:
    javascript
    // ==UserScript==
// @name         Bypass Trusted Types
// @match        https://target-site.com*
// @run-at       document-start
// @grant        none
// ==/UserScript==

if (window.trustedTypes && window.trustedTypes.createPolicy) {
    if (!window.trustedTypes.defaultPolicy) {
        window.trustedTypes.createPolicy('default', {
            createHTML: (s) => s,
            createScript: (s) => s,
            createScriptURL: (s) => s
        });
    }
}
    Используйте код с осторожностью.
    Скопировано в буфер обмена
  3. Важно: Укажите @run-at document-start, чтобы скрипт успел создать политику до того, как сайт попытается сделать это сам или выполнит блокируемую операцию. 
Вариант 4. Редактирование заголовков «на лету» (DevTools)
************** 
Для разовой проверки без установки чего-либо: 
  1. Откройте DevTools (F12) -> вкладка Network.
  2. Найдите запрос основного документа и посмотрите его заголовки (Headers).
  3. В современных версиях Chrome (вкладка Overrides) можно нажать правой кнопкой на заголовок Content-Security-Policy, выбрать Edit header и удалить часть, содержащую trusted-types. Перезагрузите страницу.

Если я использую вариант с Tampermonkey, это будет сильный удар по безопасности?

9 минут назад

Скажем прямо:
да, это заметная «дыра» в защите, но она не открывает ваш компьютер для хакеров мгновенно. 
Вот как это работает на практике: 
1. Что именно ломается?
Trusted Types — это «замок» на дверях, через которые в браузер попадает вредоносный код (XSS-атаки). Создавая пустую политику через Tampermonkey, вы: 
  • Снимаете проверку: Браузер перестает проверять, безопасна ли строка, которую сайт пытается превратить в скрипт.
  • Откатываетесь к базе: Вы возвращаетесь к уровню безопасности обычного интернета 5-летней давности, когда Trusted Types еще не существовало. 
2. Главные риски 
  • XSS-уязвимости: Если на конкретном сайте есть скрытая дыра (например, в строке поиска или комментариях), хакер сможет выполнить свой код в вашем браузере. Он сможет украсть ваши куки (сессию) или данные, которые вы вводите на этом сайте.
  • Глобальность: Если вы в Tampermonkey укажете @match *://*/* (вместо конкретного сайта), защита отключится на всех сайтах. Это уже опасно, так как один вредоносный рекламный баннер на любом ресурсе сможет атаковать вашу сессию. 
3. Насколько это критично? 
  • Для обычного пользователя: Риск умеренный. Большинство сайтов в мире до сих пор не используют Trusted Types и живут нормально. Вы просто лишаете себя «продвинутого» слоя брони.
  • Для работы с банками/криптой: Здесь риск выше. Если вы забудете выключить скрипт и пойдете в личный кабинет банка, теоретически риск перехвата данных возрастает. 
Как минимизировать удар: 
Чтобы не превращать браузер в «проходной двор», соблюдайте два правила: 
  1. Точечная настройка: В поле @match указывайте только конкретный адрес сайта, который не работает. Не используйте звездочки для всего интернета.
  2. Временное решение: Включайте скрипт в панели Tampermonkey только тогда, когда вам действительно нужно поработать с этим «сломанным» сайтом, и выключайте сразу после. 
Вердикт: Это не фатально, если скрипт ограничен одним сайтом. Это как убрать антивирусную проверку с одной конкретной папки, которой вы доверяете.

В Tampermonkey в поле @match можно указать сразу несколько адресов?
Да, конечно! В Tampermonkey вы можете добавить сколько угодно строк
@match, и скрипт будет работать только на перечисленных сайтах. 
Вот как это выглядит в коде скрипта: 
javascript
// ==UserScript==
// @name         Bypass Trusted Types Multi-site
// @match        https://example.com*
// @match        https://another-site.ru*
// @match        https://*.work-tool.net/*
// @run-at       document-start
// @grant        none
// ==/UserScript==
Используйте код с осторожностью.
Скопировано в буфер обмена
Несколько советов по заполнению: 
  1. Используйте звездочки (*):
    • https://example.com* — сработает на всех страницах этого домена.
    • https://*://* — сработает и на основном домене, и на всех поддоменах (например, ://example.com).
  2. Не смешивайте протоколы: Если сайт работает и на http, и на https, лучше добавить обе строки или использовать *://.
  3. Проверка в меню: Когда вы зайдете на сайт, иконка Tampermonkey покажет цифру «1» — это значит, что скрипт опознал адрес и включился. 
Это самый безопасный путь: вы чините только те 2-3 сайта, которые «сломаны», а на остальном интернете (включая почту и банки) защита Trusted Types остается активной.
