Вы сказали: "javascript. я делаю fetch запрос к странице некоего сайта. получаю исходный код страницы в виде текста. затем с помощью метода match() и регулярного выражения "вырезаю" оттуда нужные фрагменты и формирую из них свою собственную страницу. что лучше, оставить все как есть, или скормить полученный через fetch код domparser-у и вырезать нужное уже оттуда? вообще, как оптимизировать то что я делаю?"
Использование DOMParser почти всегда лучше, надежнее и безопаснее, чем парсинг HTML через регулярные выражения. Регулярные выражения плохо справляются с изменением структуры HTML, вложенными тегами и пробелами. 
Почему стоит перейти на DOMParser 
  • Надежность: HTML — это контекстно-зависимая грамматика. Регулярные выражения ломаются, если сайт изменит порядок атрибутов в теге или добавит класс.
  • Удобство: Вместо сложных match() вы сможете использовать привычные методы вроде querySelector() и querySelectorAll().
  • Безопасность: DOMParser создает документ в памяти без выполнения вредоносных скриптов (<script>), которые могли быть на чужой странице. 
Пример реализации с DOMParser 
javascript
fetch('https://example.com')
  .then(response => response.text())
  .then(htmlText => {
    // 1. Инициализируем парсер
    const parser = new DOMParser();
    
    // 2. Превращаем строку в DOM-документ
    const doc = parser.parseFromString(htmlText, 'text/html');
    
    // 3. Извлекаем данные с помощью стандартных селекторов
    const title = doc.querySelector('h1')?.textContent;
    const articles = doc.querySelectorAll('.article-item');
    
    // 4. Формируем свою страницу
    console.log(title);
  });
Как еще больше оптимизировать ваш процесс 
  • Фильтрация на стороне сервера (CORS-прокси): Если вы делаете запрос из браузера напрямую, вы наверняка столкнетесь с ошибкой CORS. Использование своего backend-скрипта (на Node.js) позволит парсить HTML там (например, через библиотеку cheerio) и отдавать на фронтенд уже готовый чистый JSON. Это сэкономит трафик клиента. 
  • Обработка ошибок: HTML на чужих сайтах часто бывает "битым". Метод textContent безопаснее, чем innerHTML, так как он автоматически экранирует спецсимволы и защищает от XSS-атак при вставке данных на вашу страницу. 
  • Атрибуты вместо текста: Если нужные данные лежат внутри data-атрибутов или ссылок, DOMParser позволит достать их одной командой: element.getAttribute('href'). Регулярным выражением это делать крайне неудобно. 
Использование DOMParser — идеальное решение для вашего объема данных. Главная страница новостного сайта весит в среднем 1–3 МБ, с чем браузерный DOMParser справляется за считанные миллисекунды. 
Как оптимизировать сбор статей и комментариев 
Раз вы собираете статьи (с картинками или без) и комментарии, структура HTML может быть вложенной. Вот пошаговая стратегия оптимизации: 
1. Используйте итераторы вместо тяжелых массивов 
При сборе множества элементов (например, комментариев) querySelectorAll возвращает NodeList. Для экономии памяти преобразуйте его эффективно или перебирайте напрямую. 
javascript
const parser = new DOMParser();
const doc = parser.parseFromString(htmlText, 'text/html');

// Оптимальный перебор комментариев без создания лишних массивов
const comments = doc.querySelectorAll('.comment-text');
const commentData = Array.from(comments, el => el.textContent.trim());
2. Безопасное извлечение картинок 
Для статей с картинками важно правильно собирать атрибуты, учитывая, что на новостных сайтах часто используется отложенная загрузка (lazy loading). 
javascript
const articles = doc.querySelectorAll('.news-card');

const parsedArticles = Array.from(articles).map(card => {
  const imgElement = card.querySelector('img');
  
  // Новостные сайты часто прячут реальный URL картинки в data-атрибуты
  const imgUrl = imgElement?.getAttribute('data-src') || imgElement?.src || 'placeholder.jpg';
  
  return {
    title: card.querySelector('.title')?.textContent.trim(),
    image: imgUrl,
    text: card.querySelector('.description')?.textContent.trim()
  };
});
3. Фрагменты DOM для быстрой отрисовки (DocumentFragment) 
Когда вы начнете собирать свою страницу из вырезанных кусочков, не вставляйте каждый элемент в DOM по отдельности. Это вызывает тяжелую операцию Reflow (перерисовку интерфейса). Используйте DocumentFragment как временный невидимый контейнер. 
javascript
const fragment = document.createDocumentFragment();

parsedArticles.forEach(data => {
  const container = document.createElement('div');
  container.className = 'my-article';
  
  // Безопасное наполнение текстом во избежание XSS
  const titleEl = document.createElement('h2');
  titleEl.textContent = data.title;
  
  const imgEl = document.createElement('img');
  imgEl.src = data.image;
  
  container.append(titleEl, imgEl);
  fragment.appendChild(container); // Добавляем во фрагмент в памяти
});

// Всего один цикл перерисовки для всей страницы
document.getElementById('my-page-root').appendChild(fragment);
Важное предупреждение по поводу CORS-расширений 
Хотя сейчас расширение решает проблему, этот подход сработает только в вашем браузере. Если вы планируете делиться этой страницей с другими или переносить проект на хостинг, код сломается. Для долгосрочной перспективы лучше сразу заложить в архитектуру бесплатный CORS-прокси (например, развернуть микрокод на Vercel, Cloudflare Workers или использовать готовый allorigins.win). 
Вы сказали: "А этим способом через DOMParser можно ведь захватить большой фрагмент с несколькими статьями и вставить в мою страницу целиком, со всем текстом и всеми картиками?"
Да, через DOMParser можно захватить любой крупный фрагмент (например, весь блок <main> или <div> со списком новостей) и вставить его на свою страницу целиком одной командой. 
Как вставить большой фрагмент целиком 
Вы можете использовать метод cloneNode(true), чтобы полностью скопировать узел со всем его содержимым (текстом, картинками и вложенной структурой), а затем вставить его к себе. 
javascript
const parser = new DOMParser();
const doc = parser.parseFromString(htmlText, 'text/html');

// 1. Находим большой родительский контейнер со статьями
const sourceContainer = doc.querySelector('.news-feed-container');

if (sourceContainer) {
  // 2. Клонируем его (true означает "глубокое клонирование" со всеми детьми)
  const clonedContainer = sourceContainer.cloneNode(true);
  
  // 3. Вставляем целиком на свою страницу
  document.getElementById('my-page-root').appendChild(clonedContainer);
}
⚠️ Важный нюанс с картинками при таком подходе: 
Если на оригинальном сайте ссылки на картинки были относительными (например, <img src="/images/pic.jpg">), то на вашей локальной странице они сломаются, так как ваш браузер попытается найти их на вашем компьютере. 
Чтобы этого избежать, перед вставкой клона перепишите относительные ссылки в абсолютные: 
javascript
// Находим все картинки внутри скопированного куска
clonedContainer.querySelectorAll('img').forEach(img => {
  const src = img.getAttribute('src');
  if (src && !src.startsWith('http')) {
    // Превращаем "/images/pic.jpg" в "https://site.com"
    img.src = new URL(src, 'https://site.com').href; 
  }
});

  • Нужно ли вам очищать текст от рекламы или посторонних ссылок внутри этого большого фрагмента? 
Вы сказали: "Хотелось бы сохранить стили оригинального сайта. Поэтому я думаю вставить себе head оригинального сайта целиком."
Вставить <head> оригинального сайта целиком — это рабочая идея, но она имеет несколько скрытых проблем, которые могут полностью сломать отображение или логику вашей страницы. 
С какими проблемами вы столкнетесь 
  1. Относительные ссылки на CSS: Почти все современные сайты подключают стили через пути вида <link rel="stylesheet" href="/assets/main.css">. На вашей локальной странице этот файл не найдется. 
  2. Лишние скрипты: Вместе со стилями в <head> переедут чужие JS-скрипты (метрики, аналитика, реклама). Они будут выдавать горы ошибок в консоли, пытаться заблокировать страницу или отслеживать ваши действия. 
  3. Конфликт стилей: Чужой CSS часто жестко сбрасывает базовые стили для тегов body, html, h1, p. Это исказит внешний вид ваших собственных элементов на странице. 
Как сделать это правильно и чисто 
Вместо копирования всего <head>, лучше вытащить из него только абсолютные ссылки на CSS-файлы и встроенные теги <style>. 
Вот готовое решение, которое автоматически соберет все стили, исправит относительные ссылки на абсолютные и добавит их на вашу страницу: 
javascript
const parser = new DOMParser();
const doc = parser.parseFromString(htmlText, 'text/html');
const originalSiteUrl = 'https://example.com'; // Укажите адрес донора

// Создаем контейнер, куда соберем только правильные стили
const styleFragment = document.createDocumentFragment();

// 1. Собираем внешние файлы стилей (<link rel="stylesheet">)
doc.querySelectorAll('link[rel="stylesheet"]').forEach(link => {
  const href = link.getAttribute('href');
  if (href) {
    const newLink = document.createElement('link');
    newLink.rel = 'stylesheet';
    // Превращаем относительный путь "/css/style.css" в "https://example.com"
    newLink.href = new URL(href, originalSiteUrl).href;
    styleFragment.appendChild(newLink);
  }
});

// 2. Собираем внутренние стили (<style>...</style>)
doc.querySelectorAll('head style').forEach(styleTag => {
  const newStyle = styleTag.cloneNode(true);
  styleFragment.appendChild(newStyle);
});

// 3. Безопасно добавляем очищенные стили в head вашей страницы
document.head.appendChild(styleFragment);
💡 Лайфхак: Изоляция стилей через Shadow DOM 
Если чужие стили начнут ломать оформление вашей личной страницы, вы можете изолировать оригинальный контент с помощью Shadow DOM. 
Вместо того чтобы добавлять чужие стили в общую document.head, вы можете создать закрытую "коробку" прямо внутри своей страницы: 
javascript
// Создаем контейнер-пустышку на своей странице
const container = document.getElementById('my-page-root');

// Включаем в нем изолированную зону (Shadow Root)
const shadow = container.attachShadow({ mode: 'open' });

// Теперь добавляем и чужие стили, и чужой HTML строго ВНУТРЬ shadow
shadow.appendChild(styleFragment); // Стили применятся только к контенту ниже
shadow.appendChild(clonedContainer); // Ваш скопированный блок со статьями
При таком подходе стили оригинального сайта будут работать только внутри этого блока и никак не повлияют на дизайн вашей остальной страницы.
