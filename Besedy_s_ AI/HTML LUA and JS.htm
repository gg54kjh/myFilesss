Теперь ситуация прояснилась! StrokesPlus (или StrokesPlus.net) — это не «Lua в браузере», а десктопное приложение для Windows, которое использует Lua для управления системой. Браузер здесь выступает просто как одно из окон. 
Поскольку это два разных процесса (приложение StrokesPlus и браузер), они не видят переменные друг друга напрямую. Вариант с файлом здесь возможен, но для одноразовой передачи координат в браузер есть более изящный способ — выполнение JavaScript через адресную строку или системные команды. 
Вот два самых рабочих способа для StrokesPlus: 
Способ 1: Передача через адресную строку (JS Injection) 
StrokesPlus может «вбросить» команду в браузер, имитируя ввод в адресную строку. Это сработает, если браузер сейчас в фокусе. 
В StrokesPlus (Lua): 
lua
local x = acGetMouseLocationX()
local y = acGetMouseLocationY()

-- Формируем команду, которая создаст переменную в консоли браузера
local js_code = string.format("javascript:window.luaCoords = {x: %d, y: %d}; void(0);", x, y)

-- Эмулируем нажатие Alt+D (фокус в адресную строку), вставку и Enter
acSendKeys("%d")
acDelay(50)
acSendKeys(js_code)
acSendKeys("{ENTER}")
После этого в консоли браузера (F12) будет доступен объект window.luaCoords. 
Способ 2: Через временный файл (Ваш вариант №4) 
Это самый надежный способ, так как он не мешает пользователю и не дергает адресную строку. 
1. В StrokesPlus (Lua): 
lua
local x = acGetMouseLocationX()
local y = acGetMouseLocationY()

local f = io.open("C:\\temp\\coords.js", "w")
if f then
    -- Записываем как JS-переменную
    f:write(string.format("window.luaCoords = {x: %d, y: %d};", x, y))
    f:close()
end
2. В браузере (JavaScript):
Если вы сами разрабатываете страницу, вставьте этот скрипт. Если нет — его можно запустить через консоль: 
javascript

script.src = 'file:///C:/temp/coords.js'; // Путь к файлу
document.head.appendChild(script);

// Через мгновение данные будут доступны:

Браузер вырезает из адресной строки слово javascript: и вставленный код без javascript просто отправляется в поиск Google. Но я использую Tampermonkey, если это может помочь делу.
Это классическая проблема защиты современных браузеров. Поскольку у вас есть Tampermonkey, мы можем реализовать элегантную схему через «файл-мост». 
Создайте на диске C: папку temp (или любую другую), чтобы путь был простым: C:\temp\coords.js. 
Нюанс с координатами: acGetMouseLocationX() в Strokes Plus выдает координаты относительно всего экрана. Если в браузере div появляется со смещением, используйте в JS left: ${x - window.screenX}px и top: ${y - window.screenY}px, чтобы скорректировать положение относительно окна. 
1. Настройка Strokes Plus (Lua) 
Добавьте в ваш жест код, который сначала пишет файл, а затем имитирует нажатие клавиши (например, F9): 
lua

-- 3. Жмем F9, чтобы Tampermonkey подхватил файл
acSendKeys("{F9}")
2. Настройка Tampermonkey (JavaScript)
 Это тот самый "мост" между StrokesPlus и js в браузере 
Скрипт будет ждать нажатия F9 и мгновенно загружать файл. Обратите внимание на расчет координат: я добавил вычет window.screenX/Y, чтобы div появлялся именно там, где стоит курсор на экране, даже если браузер не развернут на весь экран. 
javascript
// ==UserScript==
// @name         StrokesPlus Auto Bridge
// @match        *://*/*
// @grant        none
// ==/UserScript==

(function() {
    'use strict';

    window.createDivFromLua = function(screenX, screenY) {
        // Вычисляем координаты внутри окна браузера
        var relativeX = screenX - window.screenLeft;
        var relativeY = screenY - window.screenTop;

        var d = document.createElement('div');
        d.style.cssText = `
            position: fixed;
            left: ${relativeX}px;
            top: ${relativeY}px;
            width: 30px;
            height: 30px;
            background: rgba(255, 0, 0, 0.7);
            border: 2px solid white;
            z-index: 2147483647;
            border-radius: 50%;
            pointer-events: none; 
        `;
        document.body.appendChild(d);
        
        // Опционально: удаляем div через 3 секунды
        setTimeout(() => d.remove(), 3000);
    };

    window.addEventListener('keydown', function(e) {
        if (e.key === "F9") {
            // Предотвращаем стандартное действие F9 в браузере
            e.preventDefault();
            
            var oldScript = document.getElementById('lua-bridge');
            if (oldScript) oldScript.remove();

            var script = document.createElement('script');
            script.id = 'lua-bridge';
            script.src = 'file:///C:/temp/coords.js?t=' + Date.now();
            document.head.appendChild(script);
        }
    });
})();
А можно сделать, чтобы этот скрипт Auto Bridge сам не создавал div, а создавал бы вроде как глобальную переменную для Tampermonkeey. В которой хранятся координаты жеста. И эта переменная видна из всех скриптов. А я уже буду эту переменную использовать в разных скриптах и для разных целей.
Да, это даже более правильный подход. Мы сделаем из файла coords.js своего рода «шину данных». 
Для этого нам нужно изменить логику: файл будет не выполнять действие, а просто обновлять объект в глобальном пространстве имен window. 
1. Скрипт в Strokes Plus (Lua) 
Здесь мы меняем содержимое файла так, чтобы он просто присваивал значения объекту. 
lua
local x = acGetMouseLocationX()

if f then
    -- Записываем данные в глобальный объект

2. Скрипт-приемник в Tampermonkey (Auto Bridge) 
Этот скрипт будет отвечать только за актуализацию данных. Установите ему высокий приоритет (через @run-at document-start), чтобы он работал всегда. 
javascript
// ==UserScript==
// @name         StrokesPlus Data Bridge
// @match        *://*/*
// @grant        none
// @run-at       document-start
// ==/UserScript==

(function() {
    'use strict';

    // Создаем пустой объект заранее, чтобы другие скрипты не выдавали ошибку
    window.spLastGesture = window.spLastGesture || {x: 0, y: 0, ts: 0};

    window.addEventListener('keydown', function(e) {
        if (e.key === "F9") {
            e.preventDefault();
            
            var oldScript = document.getElementById('lua-bridge');
            if (oldScript) oldScript.remove();

            var script = document.createElement('script');
            script.id = 'lua-bridge';
            script.src = 'file:///C:/temp/coords.js?t=' + Date.now();
            
            // После загрузки скрипта данные в window.spLastGesture обновятся автоматически
            document.head.appendChild(script);
        }
    });
})();
3. Как использовать в других скриптах 
Теперь в любом другом скрипте Tampermonkey (или даже просто в консоли браузера) вы можете обращаться к window.spLastGesture. 
Пример вашего скрипта для создания div: 
javascript
// В другом скрипте
window.addEventListener('keydown', function(e) {
    if (e.key === "F9") {
        // Ждем крошечную паузу, чтобы Bridge успел обновить window.spLastGesture
        setTimeout(() => {
            const coords = window.spLastGesture;
            console.log("Используем координаты:", coords.x, coords.y);
            
            // Ваша логика создания div здесь...

            d.style.left = (coords.x - window.screenLeft) + "px";
            // и так далее...
        }, 100);
    }
});
Почему это удобно: 
  1. Централизация: Только один скрипт работает с файловой системой.
  2. Гибкость: Вы можете добавить в Lua-скрипт передачу других данных (например, имя жеста или активное окно) просто расширяя строку f:write.
  3. Доступность: Переменная window.spLastGesture видна всем скриптам на странице. 
Важный момент: Чтобы другие скрипты понимали, что данные обновились, можно добавить «событие». Хотите ли вы, чтобы Bridge-скрипт после загрузки файла рассылал CustomEvent, на который другие скрипты смогут мгновенно подписываться? 
Я плохо понимаю что это за CustomEvent, но наверное дело нужное.
CustomEvent — это способ одного скрипта «крикнуть» всем остальным: «Эй, координаты обновились, можно работать!». 
Без него вашим другим скриптам пришлось бы использовать setTimeout (гадать, когда файл загрузился) или постоянно проверять переменную по таймеру. С событием же скрипт будет срабатывать мгновенно после того, как Strokes Plus записал данные. 
Вот как это будет выглядеть: 
1. Обновленный скрипт-приемник (Data Bridge) 
Этот скрипт теперь не только обновляет переменную, но и сообщает об этом системе. 
javascript
// ==UserScript==
// @name         StrokesPlus Data Bridge
// @match        *://*/*
// @grant        none
// @run-at       document-start
// ==/UserScript==

(function() {
    'use strict';

    window.spLastGesture = {x: 0, y: 0};

    window.addEventListener('keydown', function(e) {
        if (e.key === "F9") {
            e.preventDefault();
            
            var oldScript = document.getElementById('lua-bridge');
            if (oldScript) oldScript.remove();

            var script = document.createElement('script');
            script.id = 'lua-bridge';
            script.src = 'file:///C:/temp/coords.js?t=' + Date.now();
            
            script.onload = function() {
                // Как только файл загрузился и переменная обновилась, 
                // создаем и отправляем событие "spUpdated"
                var event = new CustomEvent('spUpdated', { detail: window.spLastGesture });
                window.dispatchEvent(event);
            };
            
            document.head.appendChild(script);
        }
    });
})();
2. Как теперь будет выглядеть любой ваш другой скрипт 
Вам больше не нужно следить за нажатием клавиш или за window.spLastGesture напрямую. Вы просто «подписываетесь» на событие spUpdated. 
javascript
// ==UserScript==
// @name         My Custom Action
// @match        *://*/*
// ==/UserScript==

window.addEventListener('spUpdated', function(e) {
    // Данные находятся в e.detail
    var x = e.detail.x - window.screenLeft;
    var y = e.detail.y - window.screenTop;

    console.log("Жест зафиксирован в:", x, y);

    // Рисуем ваш div
    var d = document.createElement('div');
    d.style.cssText = `position:fixed; left:${x}px; top:${y}px; width:10px; height:10px; background:blue;`;
    document.body.appendChild(d);
});
Преимущества: 
  1. Чистый код: Основной скрипт просто работает в фоне, а вы создаете сколько угодно маленьких скриптов для разных задач.
  2. Синхронность: Вы на 100% уверены, что в момент создания div переменная уже содержит новые координаты из файла.
  3. Универсальность: В e.detail можно передать не только координаты, но и любую другую информацию, которую вы захотите дописать в Lua-скрипте.

Чтобы не копировать огромный кусок кода в каждый жест, лучше всего сделать одну общую функцию (Global Lua). 
Как это сделать правильно и красиво: 
1. Создайте общую функцию (Global Lua) 
В StrokesPlus перейдите во вкладку Global Lua и добавьте туда функцию-помощник: 
lua
function SyncToBrowser(gestureName)
    local x, y = acGetMouseLocation()
    local ctrl = acGetControlState()
    local shift = acGetShiftState()
    local alt = acGetAltState()
    
    local f = io.open("C:\\temp\\coords.js", "w")
    if f then
        f:write(string.format(
            "window.spLastGesture = {x: %d, y: %d, name: '%s', ts: %d};", 
            x, y, gestureName, os.time()
        ))
        f:close()
    end
end
2. В самих жестах пишите всего одну строку 
Теперь в настройках каждого жеста (Action) вместо старого кода пишите: 
lua
SyncToBrowser("ИМЯ_ЖЕСТА")
Например, для жеста "Влево-Вниз" напишите SyncToBrowser("LeftDown"). 
Важные моменты: 
  1. Комбинации клавиш: Если вы хотите, чтобы жест и отправлял клавиши в Windows, и уведомлял браузер, вы можете оставить обе команды:
    lua
    acSendKeys("^c") -- Скопировать (Ctrl+C)
SyncToBrowser("Copy") -- Сообщить браузеру
  2. Задержка: Браузеру нужно время, чтобы увидеть нажатие F9 и прочитать файл. Если Tampermonkey должен реагировать мгновенно, убедитесь, что ваш скрипт в Tampermonkey "слушает" именно ту клавишу, которую StrokesPlus нажимает в конце записи файла.
Теперь всё работает как единый механизм: вы рисуете жест -> StrokesPlus пишет файл и "жмет" F9 -> Tampermonkey видит F9, читает файл и выполняет ваш JS-код.
