в css есть для таблиц свойство border-collapse а есть что-то подобное для нетабличных элементов?

Алиса

Почему border-collapse работает только с таблицами

Это свойство контролирует, как будут отображаться границы между ячейками таблицы. Оно принимает два значения:

  • collapse — границы соседних ячеек объединяются в одну, что делает таблицу более компактной.
  • separate — каждая ячейка имеет собственную границу, что может приводить к появлению двойных линий в местах их соприкосновения.

Это поведение напрямую связано с внутренней логикой отображения таблиц в браузере. Для нетабличных элементов (например, <div>, <span>) нет аналогичного механизма, который бы автоматически объединял их границы при соприкосновении.

Как добиться похожего эффекта для нетабличных элементов

Если вам нужно объединить границы соседних блоков, например, чтобы создать эффект единой рамки, для нетабличных элементов используют другие подходы:

  • Обёрточные элементы. Поместите нужные блоки внутрь контейнера с display: table и задайте этому контейнеру border-collapse: collapse. Однако это меняет структуру HTML и может быть не всегда уместно.
  • Псевдоэлементы и outline. Для визуального объединения границ можно использовать псевдоэлементы :before или :after с свойством border. Также можно управлять границами через outline, задавая границу только с одной стороны элемента.
  • Комбинация border и background. Иногда эффект «общей границы» достигается за счёт того, что граница одного элемента визуально «накладывается» на фон другого.

Важный нюанс: border-spacing

Таким образом, для нетабличных элементов нет встроенного решения «из коробки», но вы можете творчески подойти к задаче, комбинируя стандартные CSS-инструменты.

А что можно вписать в content псевдоэлементу? Не в контексте темы про границы, а вообще. Только текст можно? div или span туда не вставишь?

Алиса

В свойство content для псевдоэлементов (::before, ::after) нельзя вставлять HTML‑теги вроде <div> или <span>. Браузер не будет их парсить как разметку — они либо отобразятся как обычный текст, либо (при некорректном синтаксисе) значение просто проигнорируется.

Что реально можно указать в content

  • Строка текста в кавычках: 
    ::before { content: "Привет!"; }
  • Пустая строка (чтобы псевдоэлемент существовал, но без видимого текста): 
    ::before { content: ""; }
  • Изображение через url(): 
    ::before { content: url("icon.png"); }
    Учтите: у таких изображений плохо управляются размеры стандартными width/height — часто лучше использовать background.
  • Символы Unicode (в том числе через экранирование): 
    ::after { content: "\2605"; /* ★ */ }
  • Значения атрибутов через функцию attr(): 
    a::after { content: " (" attr(href) ")"; }
  • Кавычки с помощью ключевых слов: 
    q::before { content: open-quote; }
q::after  { content: close-quote; }
    Работает в связке с quotes на родителе.
  • Счётчики (counter, counters) — удобно для кастомной нумерации: 
    li::before {
  counter-increment: my-counter;
  content: counter(my-counter) ". ";
}

Почему нельзя вставить <div> или другой тег

content — это не HTML-разметка, а генерируемый контент, который браузер отображает как анонимный строчный (или блочный, если вы зададите display) узел внутри псевдоэлемента. Он не поддерживает вложенность и теги.

Если попробовать:

::before { content: "<div>текст</div>"; }

то на странице появится именно текст <div>текст</div>, а не элемент div.

Как добиться похожего на «вложенный элемент» эффекта

Если цель — визуально оформить что‑то внутри псевдоэлемента, используют:

  • Стили самого псевдоэлемента: display, width, height, background, border, padding и т. д.
  • Несколько псевдоэлементов: у одного элемента можно одновременно использовать и ::before, и ::after, стилизуя их по‑разному.
  • Комбинацию с реальным HTML: если нужна сложная вложенность — лучше добавить нужный элемент в разметку, а не пытаться «сгенерировать» его через content.

Я просто хотел узнать в целом как и что. А зачем бывает нужно создавать пустой псевдоэлемент, просто чтоб он там был?

Алиса

Пустой псевдоэлемент (content: "") создают не ради «чтобы был», а чтобы получить встраиваемый графический объект без лишнего HTML. Сам по себе он невидим, но у него можно задать размеры, фон, границы, тени, трансформации — и использовать как декоративный или интерактивный слой.

Частые практические случаи

  • Декоративные элементы (точки, линии, уголки, маркеры). Например, полоска слева у блока цитаты или маленькая цветная метка у заголовка: 
    .quote::before {
  content: "";
  display: block;
  width: 4px;
  height: 100%;
  background: #3b82f6;
  margin-right: 12px;
}
  • Имитация сложных рамок и эффектов. Двойная рамка, скошенные углы, «вырезанные» части — всё это часто делают через пустой псевдоэлемент с фоном/границами и наложением.
  • Анимации фона и состояний. Классический пример — кнопка с эффектом «заполнения» при наведении: 
    button {
  position: relative;
  z-index: 1;
  background: transparent;
  color: #000;
}
button::before {
  content: "";
  position: absolute;
  inset: 0;
  z-index: -1;
  background: #3b82f6;
  transform: scaleX(0);
  transform-origin: right;
  transition: transform 0.3s ease;
}
button:hover::before {
  transform: scaleX(1);
}
    Здесь псевдоэлемент — это «слой», который двигается и перекрывает кнопку.
  • Тени и наложения с нестандартной формой. Если нужна тень только с одной стороны или тень в виде фигуры, проще сделать её отдельным слоем через псевдоэлемент, чем пытаться добиться этого у самого элемента.
  • Создание «отрицательного» пространства или вырезов. Например, эффект «дырки» в фоне или перекрытие части родителя, чтобы визуально «вырезать» кусок.
  • Счётчики и нумерация без текста в HTML. Псевдоэлемент может содержать только content: counter(...), а всё оформление (кружок, рамку, отступы) делают через стили пустого блока.
  • Работа с clip-path и сложными формами. Пустой псевдоэлемент удобно обрезать clip-path, чтобы получить нестандартную фигуру, не ломая верстку основного элемента.

Важные технические моменты

  • Псевдоэлемент не существует без content. Даже если он пустой, свойство content: "" обязательно — иначе браузер не создаст псевдоэлемент и стили к нему не применятся.
  • По умолчанию он строчный. Чтобы задавать ему width, height, flex и т. д., нужно явно указать display: block или inline-block (или grid/flex при необходимости).
  • Позиционирование. Часто используют связку position: relative у родителя и position: absolute у псевдоэлемента — так получают независимый слой для эффектов.

А как с помощью псевдоэлемента сделать цветной прямоугольник без текста внутри7

Алиса

Чтобы сделать цветной прямоугольник без текста с помощью псевдоэлемента, нужно:

  1. Обязательно указать content: "" — иначе псевдоэлемент не создастся.
  2. Задать ему размеры (width и height).
  3. Поставить цвет через background-color.
  4. Обычно делают блочным (display: block или inline-block) и позиционируют, как удобно.

Простой пример (прямоугольник рядом с текстом)

<div class="box">Текст рядом с прямоугольником</div>

.box {
  position: relative;    /* чтобы позиционировать псевдоэлемент относительно этого блока */
  padding-left: 120px;   /* место под прямоугольник */
}

.box::before {
  content: "";              /* обязательно */
  position: absolute;
  left: 0;
  top: 0;
  width: 100px;             /* ширина прямоугольника */
  height: 60px;             /* высота прямоугольника */
  background-color: #3b82f6;/* цвет заливки */
  /* можно добавить border, border-radius, box-shadow и т.д. */
}

Прямоугольник «под» элементом (как фон, но отдельным слоем)

Если хочется, чтобы прямоугольник был позади контента (например, для эффектов):

.card {
  position: relative;
  z-index: 1;
  padding: 20px;
}

.card::before {
  content: "";
  position: absolute;
  inset: 0;                 /* то же, что top/right/bottom/left: 0 */
  z-index: -1;              /* уводим под контент */
  background-color: #ef4444;
  border-radius: 8px;       /* опционально: скруглённые углы */
}

Здесь inset: 0 растягивает псевдоэлемент на всю ширину и высоту родителя.

Прямоугольник как отдельный «блок» (без привязки к тексту)

Иногда делают просто декоративный блок через псевдоэлемент у пустого контейнера:

<div class="rect"></div>

.rect::after {
  content: "";
  display: block;
  width: 200px;
  height: 100px;
  background: #10b981;
}

В этом случае сам .rect может быть пустым — весь прямоугольник рисуется псевдоэлементом.

Частые полезные дополнения

  • Скруглённые углы: border-radius: 8px;
  • Рамка: border: 2px solid #fff;
  • Тень: box-shadow: 0 4px 6px rgba(0,0,0,0.2);
  • Градиент вместо сплошного цвета: background: linear-gradient(45deg, #3b82f6, #8b5cf6);
Допустим псевдоэлемент, добавленный перед строкой текста. Если не указать ему display: block он пристраивается прямо в начало строки, а строка как бы сдвигается вправо. Не поломает ли это верстку страницы?

Алиса

Да, это может сломать вёрстку — но не катастрофически, а локально: только в той строке, где находится псевдоэлемент.

Что происходит технически

По умолчанию ::before и ::after — это строчные элементы (как <span>). Если у псевдоэлемента есть размеры или отступы, он:

  • занимает место в потоке текста;
  • сдвигает текст, который идёт после него;
  • может вызвать перенос строки, если не помещается;
  • влияет на высоту строки (line-height) и базовую линию.

Пример «ломающей» ситуации:

p::before {
  content: "";
  width: 50px;
  height: 20px;
  background: red;
  /* display: inline по умолчанию */
}

Здесь красный прямоугольник встанет прямо в поток перед текстом, и текст сдвинется. Если строка узкая — текст перенесётся, и высота блока изменится.

Когда это реально «ломает» вёрстку

  1. В узких контейнерах. Например, в мобильной версии или в ячейке таблицы текст может неожиданно переноситься, из‑за чего блок станет выше.
  2. При жёстких расчётах высоты. Если где‑то используется фиксированная высота или line-clamp, лишний элемент в потоке может испортить обрезку текста.
  3. В сложных строках с выравниванием. Если вы выравниваете элементы через vertical-align, строчный псевдоэлемент может «съехать» относительно остального текста.
  4. В списках и таблицах. Неожиданный сдвиг может нарушить визуальную сетку.

Как сделать, чтобы не ломало

Вариант 1: вынести из потока (самый надёжный)

Используйте абсолютное позиционирование:

.text-block {
  position: relative;
}

.text-block::before {
  content: "";
  position: absolute;
  left: -50px;      /* вынести влево, вне потока */
  top: 0;
  width: 40px;
  height: 40px;
  background: red;
}

Теперь прямоугольник не участвует в потоке: текст остаётся на месте, а псевдоэлемент просто «висит» сбоку. Это самый частый приём для маркеров, иконок и декоративных элементов.

Вариант 2: сделать inline-block и аккуратно задать отступы

p::before {
  content: "";
  display: inline-block;
  width: 20px;
  height: 20px;
  background: red;
  margin-right: 8px;   /* контролируемый отступ до текста */
  vertical-align: middle; /* выровнять по центру строки */
}

Так вы сохраняете элемент в потоке, но управляете его поведением: задаёте отступы и выравнивание, чтобы сдвиг был предсказуемым.

Вариант 3: использовать float (реже, но бывает полезно)

p::before {
  content: "";
  float: left;
  width: 30px;
  height: 30px;
  background: red;
  margin-right: 10px;
}

Псевдоэлемент уйдёт в обтекание, а текст будет обтекать его. Но с float нужно помнить про очистку потока (clear) у следующих элементов, иначе могут быть проблемы ниже по странице.

Практическое правило

  • Если элемент должен быть частью строки (например, кастомный маркер, который должен переноситься вместе с текстом) — используйте inline-block с явными отступами и vertical-align.
  • Если элемент декоративный и не должен влиять на текст (иконка, цветной блок сбоку) — лучше position: absolute.
